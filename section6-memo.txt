================================================================
セクション6：Agent Teamsの基礎 コマンド＋プロンプト集
================================================================


────────────────────────────────────────
6.1 サブエージェントの限界とAgent Teamsの登場
────────────────────────────────────────

■ サブエージェントの3つの限界
  ① エージェント同士が会話できない（PM経由のみ）
  ② 基本的に順次実行（並列不可）
  ③ タスク間の調整が難しい（PMがボトルネックに）

■ Agent Teamsの4つの構成要素
  ① チームリード     あなたのメインClaude Codeセッション。方針決定＋タスク割振り
  ② チームメイト     独立したClaude Codeインスタンス。並列に実行される
  ③ 共有タスクリスト  全員が参照できるToDoリスト。進捗の可視化
  ④ メールボックス    チームメイト同士が直接メッセージをやりとりする仕組み

■ サブエージェント vs Agent Teams 比較
  ┌──────────────────────┬──────────────────────────┬──────────────────────────────┐
  │ 項目                 │ サブエージェント         │ Agent Teams                  │
  ├──────────────────────┼──────────────────────────┼──────────────────────────────┤
  │ 関係性               │ 上司 → 部下              │ チーム                       │
  │ エージェント間通信   │ 不可（PM経由のみ）       │ 可能（メールボックス）       │
  │ 実行方式             │ 順次                     │ 並列                         │
  │ 実行環境             │ PMのコンテキスト内       │ 独立セッション               │
  │ セットアップ         │ ファイル配置のみ         │ tmux + 設定変更              │
  │ 適するタスク         │ 単純な委任、パイプライン │ 並列開発、大規模プロジェクト │
  └──────────────────────┴──────────────────────────┴──────────────────────────────┘


────────────────────────────────────────
6.2 Agent Teamsのセットアップ
────────────────────────────────────────

【ステップ1：settings.json の編集（Agent Teams有効化）】

■ 設定ファイルを開く
  code ~/.claude/settings.json

■ 追加する内容
  {
    "env": {
      "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
    }
  }

  ※ 既存の設定がある場合は env ブロック内に1行追加する
  ※ ~/.claude/settings.json（ユーザー全体）に書くのがおすすめ


【ステップ2：tmux のインストール】

■ macOS
  brew install tmux

■ Ubuntu / WSL
  sudo apt install tmux

■ バージョン確認
  tmux -V

  ※ Windowsの方はtmuxなしでもAgent Teamsは動作する（表示方法が異なる）


【ステップ3：tmux の最低限の操作（3つだけ）】

  ┌────────────────────┬──────────────────────────┐
  │ 操作               │ コマンド                 │
  ├────────────────────┼──────────────────────────┤
  │ セッション起動     │ tmux                     │
  │ セッション終了     │ exit または Ctrl + D     │
  │ ペイン間の移動     │ Ctrl + B → 矢印キー     │
  └────────────────────┴──────────────────────────┘

  ※ 画面分割はClaude Codeが自動でやる。手動操作はほぼ不要
  ※ ペインの全画面表示：Ctrl + B → z（もう一度で元に戻る）


【ステップ4：Claude Codeを再起動して動作確認】

■ Claude Codeを終了して再起動
  /exit
  claude

■ Agent Teamsの有効化を確認するプロンプト
  Agent Teamsは有効になっていますか？

■ 使えるツールの確認プロンプト
  あなたが使えるツールの中に、Team関連のツールはありますか？

  → TeamCreate, TeamAddTask, SpawnTeammate, SendTeammateMessage
    が表示されればセットアップ完了


【2つの表示モード】

  ┌────────────────────┬────────────────────────────┬────────────────────────────┐
  │ 項目               │ インプロセスモード         │ スプリットペインモード     │
  ├────────────────────┼────────────────────────────┼────────────────────────────┤
  │ 前提               │ tmux 不要                  │ tmux セッション内で起動   │
  │ 見え方             │ 1人ずつ表示                │ 全員が同時に見える         │
  │                    │ （Shift + ↓ で切替）       │                            │
  │ おすすめ場面       │ 簡単なタスク               │ チームの動きを観察したい時 │
  └────────────────────┴────────────────────────────┴────────────────────────────┘

■ スプリットペインモードの起動順（このコースで使用）
  tmux        ← まずtmuxを起動
  claude      ← tmuxの中でClaude Codeを起動


────────────────────────────────────────
6.3 最初のAgent Teams体験：リサーチチーム
────────────────────────────────────────

【起動手順】

■ tmux → Claude Code → /clear の順で準備
  tmux
  claude
  /clear


【リサーチチームのプロンプト】

■ チームへの一括指示
  Todoアプリを本格的なWebアプリに拡張することを検討しています。
  エージェントチームを作成して、3つの観点から並列に調査してください：

  チームメイト1：「Webフレームワーク調査」
  - FastAPI、Flask、Djangoの比較を行い、このプロジェクトに最適なものを提案
  - 比較観点：学習コスト、非同期サポート、エコシステム、パフォーマンス

  チームメイト2：「データベース設計」
  - Todoデータの永続化方法を調査し、SQLite vs PostgreSQLの比較と推奨スキーマを提案
  - 開発環境と本番環境の両方を考慮すること

  チームメイト3：「デプロイ戦略」
  - Docker化の方法とクラウドサービス（Render、Railway等）へのデプロイ方法を調査
  - コスト、難易度、CI/CDとの連携を考慮すること

  調査結果をまとめて、統合的なレポートを作成してください。
  Agent Teams、tmuxを使い、別々のペインで各チームメイトを動作させてください。


【観察すべき4つのポイント】

  ① チーム作成プロセス
    TeamCreate → TeamAddTask × 3 → SpawnTeammate × 3
    → tmuxが自動でペインを分割

  ② 並列実行
    3つのペインで同時に作業が進む
    各チームメイトが独立したコンテキストウィンドウを持つ

  ③ チームメイト間のメッセージング
    SendTeammateMessage で直接連絡
    （発生しない場合もある。タスクが独立していれば不要）

  ④ 結果統合
    チームリードが各チームメイトの成果物を読み取り
    → 統合レポートを作成


【生成された成果物の確認】

■ ファイル一覧を確認
  ls -la


【よくあるトラブルと対処法】

  ┌──────────────────────────────────────┬──────────────────────────────────────────┐
  │ トラブル                             │ 対処法                                   │
  ├──────────────────────────────────────┼──────────────────────────────────────────┤
  │ チームリードが自分で調査を始める     │ 「自分では調査しないでください。         │
  │                                      │  チームメイトに任せて、結果を            │
  │                                      │  待ってください。」と指示               │
  ├──────────────────────────────────────┼──────────────────────────────────────────┤
  │ チームメイトがスタックして動かない   │ 「タスクの状況を確認してください。」     │
  │                                      │ とチームリードに指示                     │
  ├──────────────────────────────────────┼──────────────────────────────────────────┤
  │ tmuxのペインが見にくい・文字が小さい │ Ctrl + B → z で全画面表示               │
  │                                      │ （もう一度で元に戻る）                   │
  ├──────────────────────────────────────┼──────────────────────────────────────────┤
  │ トークンコストが高い                 │ チームメイトは必要最小限に               │
  │                                      │ （1人 = 1コンテキストウィンドウ消費）   │
  └──────────────────────────────────────┴──────────────────────────────────────────┘


【プロンプトのコツ（Agent Teams向け）】

  - チームメイトはチームリードの会話履歴を引き継がない
    → 指示文だけで仕事内容が完結するように書く
  - 各チームメイトの役割・調査範囲・成果物を明示する
  - 番号付きでチームメイトを列挙すると、Claude が正確にチームを構成しやすい
  - 「自分では作業せずチームメイトに任せて」と明記するのも効果的

================================================================

      │
      ▼
  チームリードがチーム作成
  （TeamCreate → TeamAddTask × N → SpawnTeammate × N）
      │
      ▼
  ┌─────────────┬─────────────┬─────────────┐
  │ チームメイト1 │ チームメイト2 │ チームメイト3 │
  │ （並列実行） │ （並列実行） │ （並列実行） │
  └──────┬──────┴──────┬──────┴──────┬──────┘
         │    メッセージ交換（必要時）    │
         └──────────┬───────────────────┘
                    │ 各メンバーが結果を報告
                    ▼
  チームリードが統合レポートを作成


================================================================
サブエージェント vs Agent Teams — 体験の違い
================================================================

  ┌──────────────┬──────────────────────────────┬──────────────────────────────┐
  │ 項目         │ サブエージェント（Section5）  │ Agent Teams（Section6）      │
  ├──────────────┼──────────────────────────────┼──────────────────────────────┤
  │ 画面         │ 1画面・1ストリーム            │ 複数ペイン・同時表示         │
  │ 実行         │ 順番に1つずつ                 │ 全員が同時に作業             │
  │ 所要時間     │ 全タスク分の合計             │ 最も遅い1タスク分            │
  │ 感覚         │ 「1つのAIが順にこなす」       │ 「複数のAIが協力している」   │
  └──────────────┴──────────────────────────────┴──────────────────────────────┘
