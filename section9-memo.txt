────────────────────────────────────────
9.2 応用アイデアと発展的なトピック
────────────────────────────────────────

【応用アイデア3つ】

■ 応用①：自分のプロジェクト用サブエージェントを作る（難易度★☆☆）
  セクション4〜5の要領で、自分の仕事に特化したエージェントを設計する

  具体例：
    フロントエンド開発者 → accessibility-checker（アクセシビリティ検証）
    データ分析者         → data-validator（データ品質チェック）
    チーム開発者         → pr-reviewer（PRレビュー観点の統一）

  設計のコツ：
    「自分が繰り返しやっている作業」を洗い出す → それがエージェント化の候補

■ 応用②：CI/CDパイプラインにClaude Codeを組み込む（難易度★★☆）
  Git push → GitHub Actions → Claude Code実行 → レビュー・テスト自動化

  ユースケース：
    PRが作られたら自動でコードレビュー
    mainマージ時にドキュメントの自動更新
    リリースノートの自動生成

  キーコマンド：
    claude -p（非対話モード）
    → スクリプトからClaude Codeを呼び出せる

■ 応用③：MCPで外部ツールと連携する（難易度★★☆）
  MCP = Model Context Protocol（Anthropicが推進するオープン標準）
  Claude Code ↔ MCPサーバー ↔ 外部サービス（Slack / DB / API など）

  できること例：
    Slackのメッセージを読んでタスクを整理
    データベースに直接クエリを投げて分析
    社内APIと連携して業務を自動化


【発展的なトピック3つ】

■ 発展①：Claude Agent SDK
  プログラム（Python等）からエージェントを制御する仕組み
  Pythonコード → Agent SDK → エージェントの起動・監視・結果取得
  → Webアプリやバックエンドに組み込める
  → より複雑なワークフローの構築が可能

■ 発展②：Hooks
  エージェントの動作の前後にカスタム処理を自動で挟む仕組み
  例：ファイル書き込み前に自動バックアップ
  例：コマンド実行前にログ記録
  → パーミッション制御のカスタマイズにも使える

■ 発展③：カスタムMCPサーバー
  自分だけのツールをClaude Codeに追加する仕組み
  TypeScript / Python で構築可能
  → 社内の独自APIやデータソースをClaude Codeに接続
  → 社内業務に特化したAIワークフローの構築に


■ マルチエージェントは普遍的なスキル
  「専門家に分業させる」「コンテキストを分離する」「並列に動かす」
  → どのAIツールでも通用する設計思想
  → ツールは変わっても、考え方は残る

コース全体で学んだコマンド一覧（クイックリファレンス）
================================================================

【Claude Code 基本操作】
  claude                    Claude Codeの起動
  claude --version          バージョン確認
  claude -p "指示文"        非対話モード（スクリプトから呼び出し用）
  claude doctor             トラブル診断

【セッション管理】
  /clear                    会話のリセット（最重要の習慣）
  /exit                     Claude Codeの終了
  /login                    認証のやり直し
  /cost                     トークン使用量・推定コストの表示
  /model                    使用モデルの表示・切り替え

【モード切り替え】
  Shift+Tab                 Normal → Auto-accept → Plan → Normal の巡回

【エージェント管理】
  /agents                   サブエージェント管理メニュー

【カスタムコマンド】
  /test src/todo.py         カスタムスラッシュコマンドの実行例
  /review src/todo.py       カスタムスラッシュコマンドの実行例

【uv（Python環境管理）】
  uv init                   プロジェクト初期化
  uv add パッケージ名       パッケージ追加
  uv add --dev パッケージ名 開発用パッケージ追加
  uv run python script.py   スクリプト実行
  uv run pytest -v          テスト実行
  uv sync                   環境の再構築

【Git 基本操作】
  git init                  リポジトリ初期化
  git add .                 全ファイルをステージング
  git commit -m "メッセージ" コミット
  git diff                  変更差分の確認

【tmux（Agent Teams用）】
  tmux                      セッション起動
  exit                      セッション終了
  Ctrl + B → 矢印キー      ペイン間の移動
  Ctrl + B → z              ペインの全画面表示切替

【サーバー起動（セクション7）】
  uv run uvicorn src.main:app --reload   FastAPIサーバー起動
  http://localhost:8000/docs              Swagger UI
